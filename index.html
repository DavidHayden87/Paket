Paket

What is Paket?

Paket is a dependency manager for .NET and Mono projects, which is designed to work well with NuGet packages and also enables referencing files directly from GitHub repositories. It enables precise and predictable control over what packages the projects within your application reference. More details are in the FAQ. If you are already using NuGet for package restore in your solution then you can learn about the upgrade process in the convert from NuGet section.

How to get Paket

Paket is available as:

  • download from GitHub.com
  • as a package Paket on nuget.org

Getting Started

Specify the version rules of all dependencies used in your application in a paket.dependencies file in your project's root:

1: 
2: 
3: 
4: 

source https://nuget.org/api/v2

nuget Castle.Windsor-log4net ~> 3.2
nuget NUnit

Install all of the required packages from the specified sources:

1: 

$ paket install

The usage is to have a .paket folder in the root of your solution to install packages.

You can place paket.references files alongside your Visual Studio project files to have Paket automatically sync references for the packages noted in that file whenever an install or update takes place.

All of the files involved should be committed to your version control system.

Determine if there are package updates available:

1: 

$ paket outdated

Download updated packages; update paket.lock file and re-install to reflect and changes:

1: 

$ paket update

The paket update command will analyze your paket.dependencies file, and update the paket.lock file.
